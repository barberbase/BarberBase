## 401-on-Tap Negative Test (Both Browsers)

This test verifies that a 401 response causes the SW to update the notification with
"Session expired — open dashboard" rather than leaving the notification unchanged
(silent failure). Per spec (`16_web_push_service_worker.md`), a stale-success is a
hard FAIL.

> **How to force a 401:** Manually craft a PAT with an expired unix_expires timestamp,
> or modify the `HMAC_SECRET` on the droplet temporarily and restore it after the test.
> This test requires backend cooperation. If backend modification is not possible in
> this test run, document as DEFERRED with a tracking note.

**Chrome 401 test:**
